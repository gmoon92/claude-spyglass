## 1. 배경 및 목적

### 문제 정의
- Claude Code의 남은 과정은 블랙박스
- 어떤 도구가 호출되는지, 얼마나 걸리는지 알 수 없음
- 프롬프트 처리 과정을 이해하고 싶음

### 목표
- Claude Code의 요청/응답 흐름을 실시간으로 관찰
- 훅 기반으로 프록시 없이 가볍게 동작
- ccflare의 기능을 향후 흡수

## 4. 결정 로그 (Decision Log)

> 이 섹션은 프로젝트 진행 중 낭낭한 주요 결정사항을 기록합니다. 각 결정은 고려된 옵션, 최종 선택, 선택 이유를 포함합니다.

### 4.1 실행 방식: 글로벌 데몬

| 항목 | 내용 |
|------|------|
| **고려된 옵션** | A. **프로젝트별 인스턴스** - 각 프로젝트마다 독립적인 spyglass 서버 실행<br>B. **글로벌 데몬** - 시스템에 한 번 설치, 모든 프로젝트가 동일 인스턴스 사용 |
| **최종 결정** | **B. 글로벌 데몬** |
| **선택 이유** | - 데이터 통합 가능 (모든 프로젝트의 세션을 한 곳에서 조회)<br>- 리소스 효율성 (여러 인스턴스 중복 실행 방지)<br>- 단점: 별도 프로세스 관리 필요 → 시작/종료 명령어로 해결 |

### 4.2 데이터 소유권: 사용자 중심

| 항목 | 내용 |
|------|------|
| **고려된 옵션** | A. **프로젝트 중심** - 각 프로젝트 디렉토리에 `.spyglass/` 폴터 생성<br>B. **사용자 중심** - `~/.spyglass/`에 모든 데이터 중앙 저장 |
| **최종 결정** | **B. 사용자 중심** |
| **선택 이유** | - 한 곳에서 모든 세션 조회 가능<br>- 프로젝트 구분은 메타데이터로 처리 (project 필드로 필터링)<br>- 프로젝트 이동/삭제 시에도 데이터 보존<br>- 백업/마이그레이션이 단순화됨 |

### 4.3 설치 방식: Bun 글로벌 패키지

| 항목 | 내용 |
|------|------|
| **고려된 옵션** | A. **Docker** - 컨테이너화하여 배포<br>B. **npm/bun 글로벌 패키지** - 패키지 매니저로 설치<br>C. **Standalone Binary** - 단일 실행 파일로 배포 |
| **최종 결정** | **B. Bun 글로벌 패키지** |
| **선택 이유** | - 개발 속도: Bun이 TypeScript 네이티브 지원으로 빠른 개발 가능<br>- ccflare와 동일한 기술 스택 유지 (Bun + TypeScript + Ink)<br>- `bun link`로 개발 중에도 쉽게 테스트 가능<br>- Docker는 오버헤드, Binary는 빌드 복잡성이 단점 |

### 4.4 UI 방식: TUI 우선

| 항목 | 내용 |
|------|------|
| **고려된 옵션** | A. **Web Dashboard 우선** - 브라우저 기반 UI<br>B. **TUI 우선** - 터미널 기반 UI (Ink 사용)<br>C. **둘 다 동시 개발** |
| **최종 결정** | **B. TUI 우선** |
| **선택 이유** | - 개발자 워크플로우에 자연스럽게 통합 (터미널에서 작업 중이므로)<br>- SSH/원격 서버 환경에서도 동작<br>- 리소스 효율성 (브라우저 오버헤드 없음)<br>- lazygit, btop 등의 성공 사례<br>- Phase 2에서 필요시 Web Dashboard 추가 가능 |

### 4.5 데이터베이스: SQLite (WAL 모드)

| 항목 | 내용 |
|------|------|
| **고려된 옵션** | A. **JSON 파일** - 단순 파일 기반 저장<br>B. **SQLite** - 경량 임베디드 DB<br>C. **PostgreSQL/MySQL** - 외부 DB 서버 |
| **최종 결정** | **B. SQLite (WAL 모드)** |
| **선택 이유** | - 별도 서버 설치 불필요 (PostgreSQL/MySQL은 오버엔지니어링)<br>- JSON보다 쿼리 성능과 동시성 처리 우수<br>- WAL 모드로 읽기/쓰기 동시성 확보<br>- 단일 파일로 백업/이전 용이 |
