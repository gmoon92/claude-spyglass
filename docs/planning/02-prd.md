# spyglass PRD (Product Requirements Document)

> Claude Code 실행 과정 가시화 도구 - 토큰 누수 탐지

---

## 1. 개요

### 1.1 제품 비전

Claude Code의 블랙박스를 열어 토큰 누수 지점을 정확히 찾아내는 개발자 도구

### 1.2 핵심 가치 제안

| 가치 | 설명 |
|------|------|
| **투명성 (A)** | Claude Code의 남은 과정을 실시간으로 가시화 |
| **효율성 (B)** | 토큰 낭비 지점을 빠르게 식별하여 개발 비용 절감 |
| **인사이트 (C)** | 요청별/스킬별 토큰 사용 패턴 분석 |
| **편의성 (D)** | 터미널에서 즉시 확인, 별도 브라우저 불필요 |

### 1.3 목표 사용자

- Claude Code를 일상적으로 사용하는 개발자
- 토큰 비용을 최적화하고 싶은 팀/개인
- 서브 에이전트, 스킬, MCP 호출 구조를 분석하고 싶은 사용자

## 2. 문제 정의

### 2.1 사용자 고충

| 문제 | 구체적 상황 |
|------|------------|
| **토큰 누수 추적 불가** | 어떤 요청에서 50K 토큰이 소모되었는지 파악 불가 |
| **빠른 로그 지나침** | 실시간 로그가 스크롤되면 놓치는 정보 발생 |
| **복잡한 구조** | 서브 에이전트 → 스킬 → MCP 호출 체인 추적 어려움 |
| **ccflare 과잉** | 프록시 설정, 다중 계정 관리 등 불필요한 복잡성 |

### 2.2 해결 방식

**훅 기반 데이터 수집** → **SQLite 저장** → **TUI 실시간 가시화**

## 3. 기능 명세 (MVP Phase 1)

### 3.1 기능 우선순위

| 우선순위 | 기능 | 설명 | 예상 소요 | 사용자 가치 |
|---------|------|------|----------|------------|
| **P0** | 요청별 토큰 분석 | 각 프롬프트/도구별 토큰 소모량 상세 조회 | 3일 | ⭐⭐⭐⭐⭐ |
| **P0** | 실시간 토큰 카운터 | 현재 세션 누적 토큰 실시간 표시 | 2일 | ⭐⭐⭐⭐ |
| **P1** | 단순 알림 | 10K 토큰 초과 시 경고 | 1일 | ⭐⭐⭐⭐ |
| **P2** | 히트맵/타임라인 | 시간대별 토큰 사용량 시각화 | Phase 2 | ⭐⭐⭐ |
| **P2** | 동적 알림 | 사용자 패턴 기반 알림 | Phase 2 | ⭐⭐⭐⭐⭐ |

### 3.2 기능 상세

#### 3.2.1 요청별 토큰 분석 (P0)

**요구사항:**
- 각 API 요청의 토큰 사용량 기록
- 요청 유형 분류 (prompt, tool_call, system)
- 프로젝트/세션별 필터링
- 상위 소모 요청 TOP 10 조회

**화면:**
```
┌─────────────────────────────────────────────────────────────┐
│ 요청별 분석                                    [F2: History] │
├─────────────────────────────────────────────────────────────┤
│ Time     │ Type      │ Tool/Model    │ Tokens │ Project    │
│──────────┼───────────┼───────────────┼────────┼────────────│
│ 14:32:01 │ prompt    │ claude-sonnet │ 12,456 │ my-project │
│ 14:32:04 │ tool_call │ Read          │    234 │ my-project │
│ 14:32:05 │ tool_call │ Agent         │  5,678 │ my-project │
└─────────────────────────────────────────────────────────────┘
```

#### 3.2.2 실시간 토큰 카운터 (P0)

**요구사항:**
- 현재 세션의 누적 토큰 실시간 갱신 (1초 간격)
- 세션 시작 시간, 경과 시간 표시
- 이전 세션 대비 증감률

**화면:**
```
┌─────────────────────────────────────────────────────────────┐
│ 실시간 모니터링                                [F1: Live]   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  총 토큰: 45,234 ▲ (+12% vs 이전 세션)                     │
│                                                             │
│  [████████████████████░░░░░░░░░░] 45K / 예상 100K          │
│                                                             │
│  세션 시작: 14:30:00 (경과 2시간 15분)                      │
│                                                             │
│  최근 요청:                                                 │
│  14:32:01 Prompt: "분석해줘"  →  12,456 tokens             │
│  14:32:04 Tool:Read(...)      →     234 tokens             │
└─────────────────────────────────────────────────────────────┘
```

#### 3.2.3 단순 알림 (P1)

**요구사항:**
- 단일 요청 10K 토큰 초과 시 시각적 경고
- 알림 레벨: 🟢정상(<5K) 🟡주의(5K~10K) 🔴경고(>10K)
- TUI 상단에 알림 배너 표시

**알림 조건:**
```
IF request_tokens > 10000 THEN
  show_alert("⚠️ 토큰 과다 사용: " + request_tokens)
  set_alert_level("CRITICAL")
```

## 5. 기술 명세

### 5.1 기술 스택

| 구성요소 | 기술 | 이유 |
|---------|------|------|
| 런타임 | Bun | TypeScript 네이티브, ccflare와 동일 |
| TUI | Ink (React) | ccflare 검증, React 개발자 친화 |
| 저장소 | SQLite (WAL) | ccflare 패턴, 단일 파일 |
| 통신 | HTTP + SSE | 실시간 데이터 스트리밍 |

### 5.3 데이터 모델

```typescript
// 세션 테이블
interface Session {
  id: string;              // UUID
  project_name: string;    // 프로젝트명
  started_at: number;      // Unix timestamp
  ended_at: number | null;
  total_tokens: number;
}

// 요청 테이블
interface Request {
  id: string;
  session_id: string;
  timestamp: number;
  type: 'prompt' | 'tool_call' | 'system';
  tool_name?: string;      // tool_call인 경우
  model?: string;          // prompt인 경우
  tokens_input: number;
  tokens_output: number;
  tokens_total: number;
  duration_ms: number;
  payload?: string;        // JSON (선택적)
}
```

## 6. 비기능 요구사항

### 6.1 성능

| 항목 | 목표 | 측정 방법 |
|------|------|----------|
| 훅 실행 시간 | < 10ms | 로그 타이밍 |
| TUI 렌더링 | 60fps | 화면 갱신 주기 |
| 데이터 쿼리 | < 100ms | SQLite 쿼리 시간 |
| 메모리 사용 | < 100MB | 프로세스 모니터링 |

### 6.2 데이터 보관

| 항목 | 값 | 설명 |
|------|-----|------|
| 보관 기간 | 30일 | 자동 삭제 |
| 데이터 위치 | ~/.spyglass/spyglass.db | 사용자 중심 |
| 백업 | Phase 2에서 수동 낵ㄴㅂ로기 기능 추가 |

### 6.3 알림 설정

| 항목 | 값 | 변경 가능성 |
|------|-----|------------|
| 임계값 | 10,000 토큰 | Phase 2에서 사용자 설정 가능 |
| 알림 방식 | TUI 내 시각적 경고 | 향후 OS 알림 추가 가능 |

## 7. 개발 계획

### 7.1 개발 일정 (2주)

| 주차 | 작업 | 산출물 |
|------|------|--------|
| **1주차** | | |
| Day 1-2 | SQLite 스키마 + 저장소 구현 | storage 패키지 |
| Day 3-4 | 훅 스크립트 + 데이터 수집 | hooks/ + server API |
| Day 5 | 실시간 카운터 TUI | tui 기본 구조 |
| **2주차** | | |
| Day 6-7 | 요청별 분석 기능 | Analysis 탭 |
| Day 8 | 단순 알림 구현 | 알림 배너 |
| Day 9-10 | 통합 테스트 + 버그 수정 | MVP 완성 |

### 7.2 개발 순서

```
1. storage (SQLite) 
   → 2. hooks (데이터 수집)
   → 3. server (API)
   → 4. tui (Live 탭)
   → 5. tui (History 탭)
   → 6. tui (Analysis 탭)
   → 7. 알림 기능
```
