# doc-spec

현행 스펙 문서 관리 스킬

## 개요

프로젝트의 현행 스펙 문서를 관리합니다.

## 문서 위치

`docs/spec.md`

> `docs/planning/05-spec.md`는 초기 개발 레거시 — 수정 금지

## 사용법

### 버전 업데이트

```
@ doc-spec:version {version}
```

예시:
```
@ doc-spec:version 0.1.2
```

### 로드맵 업데이트

```
@ doc-spec:roadmap {phase-name} {status}
```

예시:
```
@ doc-spec:roadmap "Phase 8" completed
```

### 변경 이력 추가

```
@ doc-spec:changelog {version} {description}
```

예시:
```
@ doc-spec:changelog "0.1.2" "TUI ink 5.x 호환성 수정"
```

## 스펙 문서 구조

1. **개요** - 프로젝트 정보, 목표, 가치 제안
2. **시스템 아키텍처** - 전체 구조, 데이터 흐름, 컴포넌트 매트릭스
3. **데이터 모델** - ERD, 테이블 스키마, 타입 정의
4. **API 스펙** - Collect API, REST API, SSE API
5. **TUI 스펙** - 화면 구성, 탭 구성, 키보드 단축키, 컴포넌트 목록
6. **알림 스펙** - 임계값, 레벨, UI
7. **개발 스펙** - 기술 스택, 프로젝트 구조, 개발 원칙
8. **배포 스펙** - 환경변수, 실행 명령어, 데이터 위치
9. **테스트 스펙** - 테스트 구조, 실행 방법
10. **로드맵** - Phase별 계획
11. **변경 이력** - 버전 히스토리

## 버전 관리

| 버전 | 형식 | 예시 |
|------|------|------|
| Major.Minor.Patch | x.x.x | 0.1.0 |

- **Major**: 호환성 깨지는 변경
- **Minor**: 기능 추가
- **Patch**: 버그 수정
