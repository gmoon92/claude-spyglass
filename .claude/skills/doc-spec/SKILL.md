# doc-spec

프로젝트 아키텍처 문서 관리 스킬

## 개요

프로젝트 전체 수준의 아키텍처 문서를 관리합니다.
기능별 ADR/plan/tasks는 `dev-orchestrator` 또는 `doc-adr` 스킬을 사용하세요.

## 문서 위치

`docs/architecture.md`

> `docs/planning/05-spec.md`는 초기 개발 레거시 — 수정 금지

## 아키텍처 문서 구조

1. **개요** — 프로젝트 정보, 목표, 핵심 가치
2. **시스템 아키텍처** — 전체 구조, 데이터 흐름, 컴포넌트 매트릭스
3. **데이터 모델** — ERD, 테이블 스키마
4. **API 스펙** — 엔드포인트 목록
5. **개발 스펙** — 기술 스택, 프로젝트 구조, 환경변수
6. **변경 이력** — 버전 히스토리

## 버전 관리

| 버전 | 형식 | 의미 |
|------|------|------|
| Major | x.0.0 | 호환성 깨지는 변경 |
| Minor | 0.x.0 | 기능 추가 |
| Patch | 0.0.x | 버그 수정 |

## 관례

1. `Version`, `Last Updated` 항상 최신 유지
2. 날짜: `YYYY-MM-DD`
3. 커밋: `docs(architecture): 설명`
