> 평가자: 실사용자 전문가
> 점수: **4.5/10**

---

## 경쟁 도구 비교

| 도구 | 주요 기능 | spyglass 대비 차별점 |
|------|----------|---------------------|
| **ccflare** | 프록시 기반 Claude 모니터링 | 다중 계정, 로드밸런싱, 팀 공유 |
| **Claude Code 내장** | 기본 로그, 토큰 표시 | 설정 없음, 바로 사용 |
| **Langfuse** | 클라우드 LLM 모니터링 | SaaS, 팀 협업, 고급 분석 |
| **Weights & Biases** | ML 실험 관리 | 실험 추적, 아티팩트 관리 |

## 강점

### 1. 훅 기반의 가벼움
- 프록시 설정 없이 즉시 동작
- `ANTHROPIC_BASE_URL` 변경 불필요

### 2. 로컬 데이터 저장
```bash
~/.spyglass/spyglass.db  # 로컬 SQLite
```
- 민감한 프롬프트가 외부로 나가지 않음
- 데이터 주권 확보

### 3. TUI/웹 이중 인터페이스
- 터미널: 빠른 확인
- 웹: 상세 분석

## 약점/문제점

### 1. ccflare 대비 기능 부족 🔴

| 기능 | ccflare | spyglass |
|------|---------|----------|
| 다중 계정 관리 | ✅ | ❌ |
| 로드밸런싱 | ✅ | ❌ |
| 팀 단위 비용 공유 | ✅ | ❌ |
| 고급 분석 (히트맵 등) | ✅ | ⚠️ |
| 프록시 기반 자동 수집 | ✅ | ❌ |
| 설정 없음 | ❌ (프록시 설정 필요) | ⚠️ (훅 설정 필요) |

**사용자 인터뷰:**
> "ccflare 사용자로서 spyglass로 갈아탈 이유가 없습니다. ccflare는 팀 단위로 비용을 공유할 수 있고, 더 세분화된 분석이 가능합니다."

### 2. Claude Code 내장 기능과의 차별화 부족 🔴

**Claude Code가 이미 제공하는 것:**
```bash
# 토큰 사용량 (대화 하단에 표시)
# 예: "Used 1,234 tokens"

# 툴 실행 시간
claude --time  # 또는 /time 명령

# 세션 히스토리
~/.claude/sessions/  # JSONL 파일로 저장
```

**spyglass가 추가로 제공하는 것:**
- 웹 UI (Claude Code에는 없음)
- 토큰 누적 통계
- Tool별 분석

**문제:**
- 기본 제공 기능만으로도 충분한 사용자 많음
- 추가 가치가 진입장벽보다 작음

### 3. Langfuse 대비 기능 부족

| 기능 | Langfuse | spyglass |
|------|----------|----------|
| SaaS 호스팅 | ✅ | ❌ (로컬만) |
| 팀 협업 | ✅ | ❌ |
| API 통합 | ✅ | ❌ |
| 실시간 대시보드 | ✅ | ✅ |
| 비용 | 유료 | 무료 |

**문제:**
- 팀/기업 사용자는 Langfuse 선택
- 개인 사용자만 spyglass 타겟
- 개인용으로도 기능이 빈약

### 4. 차별화 포인트 불명확

```
spyglass 포지셔닝:
- "Claude Code 모니터링" → ccflare도 가능
- "토큰 누수 탐지" → 실제로는 누수 탐지 못함
- "로컬 저장" → 장점이지만 대부분 불필요
- "가벼움" → 설정 복잡도는 무거움
```

**핵심 질문:**
> "spyglass 없이 ccflare 쓰면 뭘 못하나요?"
> → "로컬 저장?" → "그게 왜 필요한데요?"
