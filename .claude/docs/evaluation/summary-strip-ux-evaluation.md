## 2. 문제점 분석

### 2.1 정보 계층 구조 문제

**현재 순서**: 볼륨 중심 (세션 → 요청 → 토큰 → 활성 → 응답시간 | 비용 → 절감 → p95 → 오류)

| 문제 | 설명 |
|------|------|
| 실시간 모니터링 미흡 | 누적 통계가 앞에 있어 실시간 상황 파악이 늦음 |
| 성능 지표 분산 | 평균 응답시간과 p95가 구분선 양쪽에 분리됨 |
| 비용-토큰 연결성 부재 | 토큰 사용량이 비용 섹션과 떨어져 있음 |

### 2.2 라벨링 일관성 문제

| 현재 상태 | 개선 필요도 |
|-----------|-------------|
| 한글 5개 + 영어 4개 혼재 | 높음 |
| 축약 정도 불일치 ("평균 응답시간" vs "err") | 중간 |
| "saved" 의미 모호 (무엇을 절약?) | 중간 |

### 2.3 시각적 우선순위 문제

| 문제 | 현재 상태 | 개선 필요도 |
|------|-----------|-------------|
| 중요도 구분 없음 | 모든 카드 동일 visual weight | 중간 |
| 경고 신호 미흡 | 오류율 상승 시 뚜렷한 피드백 부재 | 높음 |
| 그룹 구분 약함 | 구분선만으로 섹션 차이 인지 어려움 | 낮음 |

## 4. 개선 제안

### 4.1 카드 재배열 (권장)

```
[Group 1: 실시간 상태]
- 활성

[Group 2: 성능] (인접 배치)
- 평균 응답
- P95
- 오류

[구분선]

[Group 3: 볼륨]
- 세션
- 요청

[Group 4: 비용]
- 토큰
- 비용
- 절감
```

### 4.2 라벨 통일안

| 현재 | 개선안 | 변경 |
|------|--------|------|
| 총 세션 | 세션 | "총" 제거 |
| 총 요청 | 요청 | "총" 제거 |
| 총 토큰 | 토큰 | "총" 제거 |
| 활성 | 활성 | 유지 |
| 평균 응답시간 | 평균 응답 | 축약 |
| cost | 비용 | 한글화 |
| saved | 절감 | 한글화 |
| p95 | P95 | 대문자 통일 |
| err | 오류 | 한글화 |

### 4.3 시각적 개선안

| 대상 | 현재 | 개선 |
|------|------|------|
| 활성 세션 | 녹색 텍스트 | 녹색 원형 인디케이터(●) + 미세 배경 강조 |
| 오류율 > 0% | 일반 텍스트 | 빨간색 텍스트 |
| 오류율 > 1% | 빨간색 텍스트 | 카드 테두리 강조 또는 배경 틴트 |
| 그룹 구분 | 1px 구분선 | 16px gap + 섹션 라벨 (선택사항) |
