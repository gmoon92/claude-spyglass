## 4.4 민감 정보 마스킹 (Deferred)

**결정:** 드롭 — 마스킹 구현 안 함
**결정 일자:** 2026-04-20

### 근거

| 논점 | 세부 |
|------|------|
| 로컬 전용 도구 | DB는 `~/.spyglass/`에 로컬 저장, 외부 전송 경로 없음 |
| 원본 중복 저장 | `~/.claude/projects/*/transcript.jsonl`에 이미 원본 프롬프트·명령 저장됨. spyglass만 마스킹해도 원본은 남음 |
| 본인 데이터 | 사용자 본인이 쉘에 입력한 명령·프롬프트를 본인 DB에 보관하는 것은 정상 동작 |
| 디버깅 가치 | 정규식 false positive로 정상 값을 `***` 마스킹하면 로그 분석 효용 저하 |
| 파일 권한으로 99% 커버 | `chmod 700/600`으로 다른 사용자·프로세스의 읽기를 차단하면 실질 위협은 대부분 해소 |

### 재검토 트리거

다음 조건이 충족되면 마스킹 재검토 필요:

1. **팀 공유 DB 도입** — 여러 개발자가 같은 DB를 공유하거나 CI/CD 파이프라인에서 DB를 수집
2. **SaaS/원격 업로드 기능** — 대시보드를 팀 서버로 전송하는 기능 추가
3. **백업 자동 동기화** — `~/.spyglass/` 전체를 iCloud/Dropbox/Google Drive 등 외부 동기화
4. **감사(Audit) 요구사항** — 규정 준수 (SOC2, ISO 27001 등) 요구로 PII 처리 정책 필요

### 참고 (향후 구현 시 힌트)

- API 키 패턴: `sk-ant-[A-Za-z0-9_-]{20,}`, `Bearer\s+[A-Za-z0-9._-]+`
- 적용 범위는 `requests.payload` (JSON blob), `tool_detail.command` (Bash 명령)
- Opt-in 방식 권장 (`SPYGLASS_MASK_SECRETS=1` 환경변수)
- 정규식 리스트는 `~/.spyglass/mask-patterns.json`으로 외부화

## 4.5 TUI/Web 기능 중복 (Deferred)

**결정:** 아키텍처 결정 유보 — TUI 개발 중단 상태이므로 현상 유지
**결정 일자:** 2026-04-20

### 근거

| 논점 | 세부 |
|------|------|
| TUI 개발 중단 | 현재 TUI는 유지보수 모드이며 신규 기능 개발 중단. 중복 기능이 실제 부담으로 작용하지 않음 |
| 기능 분기 없음 | 양쪽이 같은 속도로 진화하지 않는 상황에서 통합은 오히려 과잉 엔지니어링 |
| 아키텍처 선택지 | 통합 vs 선택 빌드(`build:tui-only`/`build:web-only`) 결정은 TUI 개발 재개 후 판단이 타당 |

### 재검토 트리거

1. **TUI 개발 재개** — 신규 기능 요구가 들어오면 통합 여부를 먼저 결정
2. **유지보수 부담 증가** — 웹 기능 추가 시 TUI 싱크 누락이 반복되어 일관성 문제가 가시화
3. **배포 경로 단일화 요구** — Homebrew·Docker 등 단일 산출물이 필요한 시점

### 참고 (향후 결정 시 고려사항)

- **통합안:** `packages/shared-ui/` 생성 후 웹·TUI 공통 컴포넌트 추출 (React + Ink 조합)
- **선택 빌드안:** `package.json`에 `build:tui`, `build:web` 분리, 기본값은 `build:web`
- **drop TUI 안:** TUI를 deprecated 폴더로 이동, 웹만 유지 (가장 현실적이나 기존 TUI 사용자 반발 가능)
