## ADR-R1-001: 평균 응답시간 올바른 API 필드로 수정

**상태**: 결정됨

**문제**: `fetchDashboard()`에서 `d.summary?.avgDurationMs`를 사용하지만 API는 이 값을 0으로 반환.  
실제 값은 `d.requests?.avg_duration_ms` (478.5ms)에 있음.

**결정**: `d.summary?.avgDurationMs ?? d.requests?.avg_duration_ms ?? 0`으로 fallback 추가.

**이유**: 서버 버그이지만 클라이언트 fallback으로 즉시 수정 가능. (프론트엔드)

---

## ADR-R1-002: 세션 목록 미리보기 텍스트 영역 확장

**상태**: 결정됨

**문제**: `extractFirstPrompt()` 결과를 `max-width:80px`로 제한 → 실질적으로 미리보기 불가.  
현재 세션이 어떤 작업인지 ID만으로 파악 불가 → 핵심 가치 훼손.

**결정**: 세션 행 레이아웃을 2행 구조로 변경 — 상단 행(ID·상태), 하단 행(시간·미리보기).  
미리보기 텍스트 max-width 제거, 세션 행 전체 너비 활용.

**이유**: 세션 목록은 "어떤 작업을 했나" 파악이 핵심 (UX 디자이너, 제품 관리자 합의).

---

## ADR-R1-003: 세션 시작 시각 → 상대 시간 표시 추가

**상태**: 결정됨

**문제**: "오전 11:49:01" 절대 시각만 표시 → 세션이 얼마나 오래됐는지 즉각 파악 어려움.

**결정**: `fmtRelative()` 함수 추가. 절대 시각 옆에 "(2시간 전)" 형태로 병기.

**이유**: 실시간 모니터링 도구에서 "언제"보다 "얼마 전"이 더 즉각적 정보 제공 (UX 디자이너).

---

## ADR-R1-004: 세션 상세 헤더 이모지 제거

**상태**: 결정됨

**문제**: `detailProject`에 "📁 rv-iso" 하드코딩 — CLAUDE.md 이모지 사용 금지 지침 위반.

**결정**: "📁 " 접두사 제거 → "/" 구분자로 대체 또는 레이블로 구분.

**이유**: 코딩 컨벤션 일관성 (프론트엔드).
