## ADR-R4-001: 턴 목록 순서 정방향 전환 (T1 → T4)

**상태**: 결정됨

**문제**: 턴 목록이 최신(T4) → 오래된(T1) 역순으로 표시 → 대화를 역방향으로 읽는 부자연스러운 UX.

**결정**: `renderTurnView()`에서 `turns.slice().reverse()` 적용해 T1이 위로.

**이유**: 대화는 시간 순으로 읽는 것이 자연스러움. 최신 Turn은 스크롤로 확인 가능 (UX 디자이너).

## ADR-R4-002: 날짜 필터를 fetchAllSessions에 적용

**상태**: 결정됨

**문제**: "오늘" 필터 선택 시 상단 통계/요청은 필터되나 세션 목록은 전체 세션 표시 → 필터 범위 불일치.

**결정**: `fetchAllSessions()`에 `buildQuery()` 날짜 범위 적용.

**이유**: 날짜 필터는 모든 데이터 뷰에 일관성 있게 적용돼야 함 (제품 관리자, UX 디자이너 합의).

---

## ADR-R4-003: 턴 메타 OUT 토큰 조건부 표시

**상태**: 결정됨

**문제**: "IN xxx / OUT —" → OUT이 항상 "—"이면 시각적 노이즈.

**결정**: `tokOut > 0`일 때만 "/ OUT xxx" 추가. 없으면 "IN xxx"만 표시.

**이유**: 불필요한 "—" 제거로 정보 밀도 향상 (프론트엔드).

---

## ADR-R4-004: 세션 상세 토큰 레이블 한국어 통일

**상태**: 결정됨

**문제**: "60.2k tokens" → 영어 "tokens" 사용.

**결정**: "60.2k 토큰"으로 변경.

**이유**: UI 전체 언어 일관성 (한국어) (UX 디자이너).
