## ADR-R5-001: 프로젝트 토큰 바 최소 너비 보장

**상태**: 결정됨

**문제**: 소량 토큰 프로젝트(83, 45)의 bar-fill width가 0%로 렌더 → 시각적으로 미표시.

**결정**: `Math.max(1, Math.round(pct))%` 적용으로 최소 1% 너비 보장.

**이유**: 데이터가 있는 프로젝트는 반드시 시각적 표시가 있어야 함 (UX 디자이너).

---

## ADR-R5-002: fmtTimestamp 날짜 포함 개선

**상태**: 결정됨

**문제**: `fmtTimestamp()`가 시:분만 표시 → 오늘이 아닌 데이터에서 날짜 정보 누락.

**결정**: 오늘이 아닌 경우 날짜 prefix 추가. `fmtDate()`의 날짜 분기 로직 재사용.

**이유**: 필터 범위가 "이번주"로 확장될 때 날짜 없이는 언제인지 파악 불가 (프론트엔드).

---

## ADR-R5-003: OUT 컬럼 헤더 미수집 힌트 추가

**상태**: 결정됨

**문제**: OUT 컬럼이 항상 "—" → 데이터 수집 미지원인지 아닌지 불명확.

**결정**: `<th title="출력 토큰 (현재 미수집)">OUT</th>` tooltip 추가.

**이유**: 사용자 혼란 방지. 컬럼 유지는 향후 수집 시 호환성 확보 (프론트엔드).

---

## ADR-R5-004: 세션 상세 토큰 레이블 "입력 N토큰" 명시

**상태**: 결정됨

**문제**: "60.2k 토큰"이 입력 토큰인지 총 토큰인지 불명확.

**결정**: "입력 60.2k" 형식으로 변경 (compact 레이블).

**이유**: 출력 토큰 미수집 상황에서 "입력"임을 명시해야 혼란 없음 (UX 디자이너, 제품 관리자).
