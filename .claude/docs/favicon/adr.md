## ADR-001: SVG 파일 기반 단일 파비콘 채택

### 상태
**결정됨** (2026-04-18)

### 배경
현재 `index.html`에 파비콘 `<link>` 태그가 없어 브라우저가 `/favicon.ico` 404를 요청한다. 빌드 파이프라인이 없는 정적 단일 HTML 구조에서 간단하고 유지보수 가능한 파비콘이 필요하다.

### 고려한 옵션

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| SVG 단일 파일 | `packages/web/favicon.svg` + `<link>` | 벡터, 단일 소스, ETag 캐시 | Safari 구버전 미지원 |
| SVG + PNG 멀티포맷 | 복수 해상도 파일 제공 | 호환성 | 빌드 파이프라인 없어 수작업 |
| data URI inline | HTML 내 인라인 | HTTP 요청 없음 | 유지보수성 최악 |

### 결정
`packages/web/favicon.svg` 외부 파일 + `<link rel="icon" type="image/svg+xml" href="favicon.svg">` 방식 채택. 개발자 도구 특성상 Chrome/Firefox 최신 버전이 주 사용 환경이므로 SVG 단일로 충분하다.

### 이유
1. 벡터 단일 소스로 16×16 ~ 512×512 모든 크기 대응
2. git 이력 추적 가능 (3회 반복 설계 이력 보존)
3. 빌드 파이프라인 없는 현재 구조에 가장 적합

## ADR-002: 아이콘 형태 — 렌즈+크로스헤어 조준경(Reticle) 메타포

### 상태
**결정됨** (2026-04-18)

### 배경
"spyglass(망원경)" 컨셉을 16×16px 탭 파비콘에서 즉각 인식 가능하게 표현해야 한다. 두 전문가가 다른 형태를 제안했다.

### 고려한 옵션

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| 렌즈 원 + 손잡이 대각선 | 돋보기 side-view | 직관적 망원경 형태 | 16px에서 손잡이 디테일 소실 |
| 조준경(reticle) | 원 + 크로스헤어 | 16px 가독성 최우수, 모니터링 컨셉 강화 | 망원경보단 조준경 이미지 |
| 이니셜 텍스트 | "CS" 텍스트 | 브랜드 직접 노출 | 폰트 의존성, 16px 비가독 |

### 결정
**조준경(reticle) 메타포** 채택: 외곽 링 + 크로스헤어 + 중앙 점. 3회 반복에서 점진적 정제.

### 이유
1. 프론트엔드 관점: 16px 렌더링 시 원형 중심 구조가 인식률 최고
2. 아키텍트 관점: 모니터링/감시 컨셉을 "조준" 메타포가 직접 전달
3. 복잡한 손잡이 디테일보다 단순 geometry가 소형에서 유리

### 전문가 이견
**아키텍트**: 렌즈+손잡이가 spyglass를 더 직접 표현  
**프론트엔드**: 16px 가독성에서 reticle이 압도적 우위  
**해소**: 최종 목적이 "브라우저 탭 식별성"이므로 가독성 우선 → reticle 채택

## ADR-003: 다크/라이트 테마 대응 전략

### 상태
**결정됨** (2026-04-18)

### 고려한 옵션

| 옵션 | 설명 |
|------|------|
| 완전 다크 배경 고정 | `#0f0f0f` 배경 원/사각 |
| `prefers-color-scheme` SVG 내 미디어쿼리 | 다크/라이트 분기 |
| 투명 배경 + accent 아이콘 | 탭 배경에 블렌딩 |

### 결정
SVG 내부 `<style>` 태그에 `@media (prefers-color-scheme: light)` 삽입. 기본은 다크 배경(`#1a1a1a`), 라이트 모드에서 배경을 밝게 전환.

### 이유
- Chrome 79+/Firefox 67+ 지원으로 주 사용자 커버
- 단순 accent 색상은 다크 탭에서 가시성 부족할 수 있음
- 배경 원(rx 둥근 사각) + foreground 아이콘 구조가 any 탭 배경에서 일관성 제공

## ADR-004: 3회 반복 개선 버전 관리

### 상태
**결정됨** (2026-04-18)

### 결정
- `favicon.svg` — 현재 최신 파일 (항상 덮어씀)
- `favicon-v1.svg`, `favicon-v2.svg`, `favicon-v3.svg` — 각 이터레이션 이력 보존
- `index.html` `<link>`는 항상 `favicon.svg` 참조 (캐시 버스팅 `?v=N`)

### 이유
- 반복 이력을 파일로 보존하여 비교 가능
- 최종 배포 시 `favicon.svg` 단일 파일만 필요
