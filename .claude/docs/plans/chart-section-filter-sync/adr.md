## ADR-001: timeline-meta 라벨 SSoT 캡슐화

### 상태
**결정됨** (2026-04-27)

### 배경

`#timelineMeta` 안의 두 그룹 (`품질` / `누적 볼륨`)에는 `<span class="timeline-meta-group-label">` 라벨이 각각 존재한다. dateFilter("전체"/"오늘"/"이번주") 변경 시 두 라벨이 즉시 갱신되어야 하는데, 갱신 로직이 클릭 핸들러·초기화·향후 SSE 재로드 등 여러 곳에 산재할 경우 단일 책임 위반과 라벨-데이터 불일치 회귀 위험이 발생한다.

기존 코드 (`packages/web/assets/js/main.js:329`)는 `#chartSubtitle`만 인라인 객체로 매핑했고, timeline-meta 라벨은 아예 갱신 로직이 없어 하드코딩된 "지난 30분" / "오늘"이 백엔드 데이터와 거짓 정보로 충돌하고 있다.

### 고려한 옵션

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| A | 인라인 분기 (subtitles 매핑처럼 호출처마다 객체 정의) | 단순 | 갱신처 늘면 누락 위험 / SSoT 위배 |
| B | `RANGE_LABELS` 상수 + `applyRangeLabels(range)` 함수 캡슐화 | SSoT 한 곳 / 호출처 1줄 / i18n 확장 용이 | 함수 1개 추가 |
| C | 컴포넌트화 (Web Component / 별도 모듈) | 재사용성 최대 | 본 규모에선 과설계 |

### 결정

**옵션 B 채택.**

- `RANGE_LABELS` 객체를 `main.js` 모듈 스코프 상수로 정의
- `applyRangeLabels(range)` 함수 도입 — `RANGE_LABELS[range]`를 조회해 두 `.timeline-meta-group-label`에 적용
- 호출처 2곳: dateFilter 클릭 핸들러 / 초기화 시점

### 이유

1. **Java 캡슐화 원칙 준수** — 동일 판단 로직은 한 곳에만 (CLAUDE.md 개발 원칙).
2. **SSoT** — 라벨 텍스트 변경 / i18n 도입 시 단일 지점 수정.
3. **확장성** — SSE 재로드, URL 파라미터 동기화 등 추가 호출처가 생겨도 1줄로 적용 가능.
4. **현재 산재된 패턴 (subtitles)을 같은 모양으로 정리** → 코드 일관성.

## ADR-002: chartSubtitle 의미 정리 — "최근 30분 · 실시간" 고정

### 상태
**결정됨** (2026-04-27)

### 배경

`#chartSubtitle`은 `view-section-header` 안에서 `요청 추이 (실시간)` 패널 라벨의 보조 부제로 작동한다. 정작 부제가 가리키는 `timelineChart`는 클라이언트 sliding 30분 윈도우 차트로, date-filter ("전체"/"오늘"/"이번주")와 본질적으로 무관하다.

그러나 현재 `main.js:329-333`의 `subtitles` 객체가 dateFilter 변경 시 부제를 "전체 기간"/"오늘"/"이번 주"로 바꿔 → **사용자 혼란 발생** ("이 차트가 전체 기간을 보여주나?" 오해).

### 고려한 옵션

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| A | "최근 30분 · 실시간"으로 고정. subtitles 매핑/갱신 삭제 | 차트 본질 정확 반영 / 의미 충돌 제거 | 부제 정보가 정적 |
| B | `#chartSubtitle` 제거 | 중복 제거 (시각축에 이미 시간 존재) | "실시간" 본질이 시각화에서 즉시 인지 안 될 수 있음 |
| C | 현행 유지 (필터별 텍스트) | 변경 없음 | 데이터-라벨 의미 충돌 방치 |

### 결정

**옵션 A 채택.**

- `index.html` `#chartSubtitle` 초기값을 `"최근 30분 · 실시간"`으로 변경
- `main.js` `subtitles` 객체와 그 갱신 코드 삭제

### 이유

1. **본질 일치** — timelineChart는 30분 sliding 윈도우 + 실시간 SSE 갱신이므로 "최근 30분 · 실시간"이 정확.
2. **정보 위계 분리** — timeline-meta는 필터 윈도우 컨텍스트, chartSubtitle은 차트 본질. 두 컨텍스트가 분리되어 사용자 혼란 제거.
3. **코드 단순화** — subtitles 매핑/갱신 삭제로 핸들러 가독성 향상.

## ADR-003: timeline-meta 그룹 라벨 표기 방식 — "본질 · 범위"

### 상태
**결정됨** (2026-04-27)

### 배경

`#timelineMeta`의 두 그룹은 본질이 다르다.

- 그룹 1 (`품질`): 평균 / P95 / 오류율 — 요청 품질을 평가
- 그룹 2 (`누적 볼륨`): 세션 / 요청 / 토큰 — 누적 사용량

활성 date-filter 윈도우만 라벨로 표시하면 두 그룹의 본질 구분이 흐려지고, 본질만 표시하면 데이터-라벨 불일치가 재발한다.

### 고려한 옵션

| 옵션 | 설명 | 예시 | 장점 | 단점 |
|------|------|------|------|------|
| 1 | 활성 범위만 노출 | "[전체 기간] 평균 P95 오류율" / "[전체 기간] 세션 요청 토큰" | 단순 | 두 그룹의 의미 구분 약화 |
| 2 | 본질 + 범위 결합 | "[품질 · 전체 기간] 평균 P95 오류율" / "[누적 · 전체 기간] 세션 요청 토큰" | 정보 풍부 / 그룹 본질 유지 | 라벨 폭 약간 증가 |

### 결정

**옵션 2 채택.**

- 라벨 형식: `<본질> · <범위 라벨>`
  - 그룹 1: `품질 · {범위}` (예: "품질 · 전체 기간")
  - 그룹 2: `누적 · {범위}` (예: "누적 · 전체 기간")
- `applyRangeLabels(range)` 함수가 본질 prefix를 합성
- `aria-label`도 같은 패턴: "요청 품질 (전체 기간)" / "누적 볼륨 (전체 기간)"

### 이유

1. **본질 + 컨텍스트** — 그룹 정체성(품질/누적)을 유지하면서 활성 윈도우를 명시.
2. **시각 위계** — `<본질>`이 그룹의 고정 정체성, `<범위>`가 컨텍스트 보조 — 인지 부담 없이 두 정보 병기.
3. **가독성** — 구분자 `·`(middle dot)는 기존 timeline-meta 스타일과 일관 (다른 곳에서 이미 사용됨).
4. **i18n 확장성** — 본질/범위 두 토큰을 별도 매핑으로 관리 가능.

### 영향

- `index.html`: 두 `.timeline-meta-group-label` 초기값 갱신 + 두 그룹 `aria-label` 패턴 갱신
- `main.js`: `RANGE_LABELS` 상수 / `applyRangeLabels(range)` 함수 / 클릭 핸들러 / 초기화에서 호출
- CSS: 변경 없음 (기존 `.timeline-meta-group-label` 스타일 유지)
