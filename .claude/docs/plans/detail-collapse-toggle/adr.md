## ADR-001: 버튼 그룹 구조 분리

### 상태
**결정됨** (2026-04-19)

### 배경
현재 `#btnCloseDetail` 단일 버튼이 "접기"라는 라벨을 달고 있지만 실제로는 `closeDetail()`을 호출해 defaultView로 전환하는 **닫기** 동작만 수행한다. 진짜 접기(콘텐츠 숨김, 헤더만 유지)를 추가하려면 두 기능을 명확히 분리해야 한다.

### 고려한 옵션

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| A: 단일 버튼 상태 전환 | 접기 클릭 → 접힘, 재클릭 → 펼침, 별도 닫기 없음 | 버튼 수 최소화 | 닫기 경로 불명확, UX 혼동 |
| B: 두 버튼 분리 (채택) | 토글 버튼 + 닫기 버튼을 `.detail-actions` 그룹으로 묶음 | 역할 명확, 닫기 항상 접근 가능 | 버튼 2개 공간 필요 |
| C: 드래그 핸들 방식 | 헤더를 드래그해서 높이 조절 | 유연한 크기 조절 | 구현 복잡, 이 기능 범위 초과 |

### 결정
옵션 B 채택. `.detail-actions` 컨테이너에 `#btnToggleDetail`(.btn-toggle)과 `#btnCloseDetail`(.btn-close)을 나란히 배치하고 구분선으로 시각적으로 분리한다.

### 이유
1. 접기와 닫기는 의미가 다름 — 혼용하면 "접었는데 사라짐" 등의 UX 혼동 발생
2. 닫기 버튼은 접힌 상태에서도 항상 접근 가능해야 함
3. 버튼 그룹 패턴은 기존 디자인 시스템과 일관성 유지

## ADR-002: collapsed 상태를 CSS 클래스로 제어

### 상태
**결정됨** (2026-04-19)

### 배경
접기 상태를 표현하는 방법으로 JS에서 직접 요소의 style.display를 제어하거나, CSS 클래스를 토글하거나, height 트랜지션을 사용하는 방법이 있다.

### 고려한 옵션

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| A: JS style.display 직접 제어 | 각 요소마다 JS로 display 값 설정 | 명시적 | JS가 스타일 SSoT 침범, 유지보수 어려움 |
| B: CSS 클래스 토글 (채택) | `.detail-collapsed` 클래스 하나로 모든 자식 숨김 | CSS가 SSoT, JS는 클래스만 토글 | — |
| C: height:0 + overflow:hidden 트랜지션 | 부드러운 애니메이션 | 시각적으로 매끄러움 | height:auto → 0 트랜지션 불안정, 콘텐츠 높이 가변 |

### 결정
옵션 B 채택. `detailView.classList.toggle('detail-collapsed')`로 클래스를 토글하고, CSS에서 `.detail-collapsed` 하위 요소들을 `display:none` 처리한다.

### 이유
1. CSS가 렌더링 로직의 SSoT — JS는 상태만 알고 스타일은 CSS가 결정
2. 새로운 콘텐츠 영역이 추가될 때 CSS만 수정하면 됨
3. height 트랜지션은 콘텐츠 높이가 동적인 환경에서 불안정

## ADR-003: 토글 아이콘 인라인 SVG chevron 사용

### 상태
**결정됨** (2026-04-19)

### 배경
접기/펼치기 상태를 나타내는 아이콘이 필요하다. 유니코드 문자, 이미지, 인라인 SVG 중 선택해야 한다.

### 고려한 옵션

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| A: 유니코드 (⌃/⌄/▲/▼) | HTML 엔티티 사용 | 간단 | 폰트 렌더링 불일치, rotate 시 레이아웃 흔들림 |
| B: 인라인 SVG (채택) | `<svg>` chevron-down 패스 | currentColor 상속, rotate 정확, 픽셀 퍼펙트 | HTML 약간 장황 |
| C: 외부 아이콘 폰트 | Font Awesome 등 | 풍부한 선택지 | 외부 의존성 추가, 로드 지연 |

### 결정
옵션 B 채택. `viewBox="0 0 12 12"` 인라인 SVG chevron-down을 사용하고, `.detail-collapsed` 상태에서 `.btn-toggle svg`에 `transform: rotate(180deg)`를 적용한다.

### 이유
1. `currentColor`로 CSS `color` 값을 자동 상속 — hover 색상 변경 별도 처리 불필요
2. `transform: rotate` 트랜지션이 정확하고 부드러움
3. 외부 의존성 없이 완결

## ADR-004: 접힌 상태에서 헤더 클릭으로 펼치기

### 상태
**결정됨** (2026-04-19)

### 배경
접힌 상태에서 사용자가 바 형태의 헤더를 클릭했을 때 자연스럽게 펼쳐지길 기대한다.

### 고려한 옵션

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| A: 토글 버튼만 클릭 가능 | 버튼만 반응 | 명확한 클릭 타겟 | 접힌 헤더 빈 공간 클릭 시 무반응 — UX 불편 |
| B: 헤더 전체 클릭 (채택) | `.detail-collapsed .detail-header` 클릭 시 펼치기 | 자연스러운 UX, 넓은 클릭 타겟 | 닫기 버튼과 충돌 방지 필요 |

### 결정
옵션 B 채택. `.detail-collapsed` 상태에서 `.detail-header` click 이벤트가 발생하면 `toggleDetailCollapse()`를 호출한다. 단, `e.target.closest('.detail-actions')` 확인으로 닫기 버튼 클릭 시 중복 호출을 방지한다.

### 이유
1. 접힌 상태의 헤더는 클릭 가능하다는 시각적 단서(cursor:pointer)를 제공하므로 클릭 반응이 있어야 자연스러움
2. 이벤트 버블링을 활용하면 별도 이벤트 리스너 추가 없이 처리 가능

## ADR-005: 닫기 버튼 시각 스타일

### 상태
**결정됨** (2026-04-19)

### 배경
닫기 버튼은 접기 토글과 명확히 구분되어야 하며, 닫기가 "파괴적 액션"임을 시각적으로 암시해야 한다.

### 결정
- 라벨: 텍스트 없이 `✕` 아이콘만 사용
- 기본: `var(--text-dim)` 색상
- hover: `var(--red)` 색상 + `var(--red-dim)` 배경 — 파괴적 액션 경고
- 접기 토글 hover: `var(--accent)` 색상 + `var(--accent-dim)` 배경

### 이유
1. 색상 대비로 두 버튼의 역할 차이를 직관적으로 전달
2. 기존 디자인 토큰(`--red`, `--red-dim`, `--accent`, `--accent-dim`) 재사용

## ADR-006: 버튼 구분선 separator

### 상태
**결정됨** (2026-04-19)

### 배경
두 버튼이 나란히 놓일 때 시각적으로 분리되지 않으면 한 그룹처럼 보일 수 있다.

### 결정
두 버튼 사이에 `width: 1px; height: 14px; background: var(--border)` 요소(`.detail-actions-sep`)를 삽입한다.

### 이유
1. 명확한 시각적 분리 — 역할이 다른 두 버튼을 구분
2. 기존 `--border` 토큰 재사용으로 디자인 일관성 유지
