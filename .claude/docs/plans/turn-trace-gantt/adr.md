# turn-trace-gantt ADR

## ADR-001: CSS/HTML 레이어만 작성, JS 렌더링 로직 제외

### 상태
**결정됨** (2026-04-19)

### 배경
세션 상세 뷰에 간트 탭을 추가하는 작업은 두 관심사로 분리된다.
1. 탭 UI 구조(HTML)와 스타일(CSS) — 정적 레이아웃
2. Canvas 렌더링 로직(JS) — 데이터 시각화

이번 feature 범위는 1번(CSS/HTML)만 포함하며, JS 로직은 별도 단계에서 처리한다.

### 고려한 옵션

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| A | CSS/HTML 먼저, JS 나중 | 레이아웃 선확정, 역할 분리 명확 | JS 없이 탭 기능 미완 |
| B | CSS/HTML/JS 동시 작성 | 기능 완결 | 범위 초과, 서버 의존성 검토 필요 |

### 결정
옵션 A 채택 — CSS/HTML 레이어만 작성한다.

### 이유
1. 요청 명세가 CSS/HTML 작업만 명시적으로 지정함
2. 서버/DB/TypeScript 파일 수정 금지 원칙을 준수
3. 레이아웃이 먼저 확정되어야 JS 렌더러가 올바른 DOM 타겟을 사용할 수 있음

## ADR-002: CSS 변수 전용, 하드코딩 색상 없음

### 상태
**결정됨** (2026-04-19)

### 배경
프로젝트 전체가 `design-tokens.css` 의 CSS 변수 시스템을 사용한다.
범례 dot의 background 색상은 JS가 동적으로 inline style로 주입하므로 CSS에서는 지정하지 않는다.

### 고려한 옵션

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| A | CSS 변수만 사용 | 테마 일관성, 다크모드 자동 지원 | 변수명 파악 필요 |
| B | 하드코딩 색상 일부 허용 | 빠른 작성 | 테마 깨짐, 유지보수 어려움 |

### 결정
옵션 A 채택 — 모든 색상을 CSS 변수로 참조한다.

### 이유
1. `design-tokens.css` 가 다크/라이트 모드를 일괄 관리하므로 하드코딩 시 테마 깨짐
2. 디자이너 에이전트 성공 기준 항목 "CSS 변수만 사용 (하드코딩 색상 없음)" 준수

## ADR-003: Canvas 높이를 JS에서 동적 설정

### 상태
**결정됨** (2026-04-19)

### 배경
Gantt 차트의 행 수는 세션마다 다르므로 CSS에서 고정 높이를 지정하면 내용이 잘리거나 과도한 여백이 생긴다.

### 고려한 옵션

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| A | CSS height 고정 | 구현 간단 | 데이터에 따라 레이아웃 깨짐 |
| B | JS에서 행 수 × rowHeight 계산 후 동적 설정 | 데이터 기반 정확한 높이 | CSS에서 height 지정 불필요 |

### 결정
옵션 B 채택 — `#turnGanttChart` 의 height 는 CSS에서 지정하지 않고 JS가 동적 설정한다.
CSS는 `width: 100%` 와 `display: block` 만 지정한다.

### 이유
1. Gantt 차트는 데이터 행 수에 따라 크기가 결정되어야 올바른 스크롤 동작이 가능함
2. `.gantt-scroll` 이 `overflow-y: auto` 를 담당하므로 Canvas는 자신의 실제 높이만 보고하면 됨
