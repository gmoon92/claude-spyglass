## 결정 사항

### ADR-R8-001: 스켈레톤 행 표준화 함수

**결정**: `makeSkeletonRows(cols, count=2)` JS 함수를 만들어 모든 로딩 상태에서 일관되게 사용.

```js
function makeSkeletonRows(cols, count = 2) {
  const row = `<tr><td colspan="${cols}" class="table-empty"><span class="skeleton"></span></td></tr>`;
  return row.repeat(count);
}
```

- 기본 count: 2 (프로젝트 패널, 세션 패널, 툴 통계 모두 동일)
- 전역 요청 피드: 3개 (더 많은 데이터 예상)

### ADR-R8-002: 빈 상태 CSS 클래스 통일

**결정**: `.empty-hint`와 `.table-empty`를 `.table-empty` 하나로 통일. `.empty-hint`는 제거.

**이유**: 두 클래스가 동일한 스타일(padding, text-align, color, font-size). 유지할 이유 없음.

### ADR-R8-003: 툴 통계 `비율` 컬럼 헤더 명시화

**결정**: 툴 통계 `비율` 컬럼 헤더를 `호출 비율`로 변경. 

**이유**: 호출 수 기준 비율임을 명시. 토큰 기준으로 오해 방지.

### ADR-R8-004: 프로젝트 선택 스켈레톤 count → 2

**결정**: 프로젝트 클릭 시 세션 패널 스켈레톤을 3개 → 2개로 통일.

**이유**: 스켈레톤 행 표준화에 맞춤.
