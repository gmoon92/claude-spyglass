## ADR-R2-001: 세션 패널 헤더에 선택된 프로젝트명 표시

**상태**: 결정됨

**문제**: 세션 패널 hint가 "10개"만 표시 → 어느 프로젝트의 세션인지 패널 헤더에서 바로 알 수 없음.

**결정**: `sessionPaneHint` → "rv-iso · 10개" 형식으로 변경. 미선택 시 "프로젝트를 선택하세요" 유지.

**이유**: 세션 패널은 선택된 프로젝트에 종속된 컨텍스트임을 명시해야 함 (UX 디자이너).

## ADR-R2-002: 최근 요청 테이블 시각 → 상대시간 병기

**상태**: 결정됨

**문제**: "오후 12:21:04" 절대 시각만 표시 → 요청이 얼마 전인지 즉각 파악 어려움.  
실시간 모니터링 도구에서 "얼마 전"이 더 핵심 정보.

**결정**: `fmtDate()` 반환값 옆에 `fmtRelative()` 결과를 dim 색으로 병기.  
형식: `"오후 12:21 · 2분 전"`

**이유**: 실시간성 강조가 핵심 가치 (제품 관리자, UX 디자이너 합의).

---

## ADR-R2-003: 세션 목록 로딩 스켈레톤 즉시 표시

**상태**: 결정됨

**문제**: 프로젝트 선택 후 세션 목록 fetch 중 "—"이 보였다가 채워지는 순간이 있음.

**결정**: `selectProject()` 호출 즉시 스켈레톤 행 표시, 데이터 로드 완료 시 교체.

**이유**: 즉각적 피드백 제공 → Nielsen 휴리스틱 #1 충족 (UX 디자이너).

---

## ADR-R2-004: 세션 상세 헤더 UUID 단축 표시

**상태**: 결정됨

**문제**: 세션 상세 헤더에 full UUID 표시 → 공간 낭비, 가독성 저하.

**결정**: 앞 8자리 + "…" 표시. full UUID는 `title` tooltip으로 확인 가능.

**이유**: 세션 ID 정보는 이미 왼쪽 패널에서 선택하므로 상세 헤더에서는 축약해도 무방 (프론트엔드).

---

## ADR-R2-005: getSessionsByProject에 first_prompt_payload 추가 (서버)

**상태**: 결정됨

**문제**: `/api/projects/:name/sessions` API가 `first_prompt_payload`를 반환하지 않아 세션 미리보기 불가.

**결정**: `getSessionsByProject()` 쿼리에 `getAllSessions()`와 동일한 first_prompt 서브쿼리 추가.

**이유**: 세션 목록에서 "이 세션이 무엇을 했나"를 파악하는 핵심 기능 (제품 관리자).
