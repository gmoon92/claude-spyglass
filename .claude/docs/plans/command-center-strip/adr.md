## ADR-001: 구분선 구현 방식

### 상태
**결정됨** (2026-04-19)

### 배경
좌측 3개(활동 지표: 세션·요청·토큰)와 우측 4개(비용·성능 지표) 사이를 시각적으로 분리해야 한다.
summary-strip은 `display: flex`이므로 구분선을 flex 자식 요소로 삽입하는 방식과
CSS border/gap만으로 처리하는 방식 중 선택이 필요하다.

### 고려한 옵션

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| A | `<div class="stat-divider">` HTML 요소 삽입 | 마크업에서 분리 위치가 명확, flex-item으로 높이 자동 맞춤 | HTML 추가 |
| B | `stat-card:nth-child(5)::after` pseudo-element | HTML 무변경 | nth-child 번호가 지표 추가 시 깨짐, 유지보수 취약 |
| C | `border-left` on first new stat-card | 단순 | 구분선 높이·여백 조절 불편 |

### 결정
옵션 A — `.stat-divider` 요소를 HTML에 삽입하고 CSS로 스타일링한다.

### 이유
1. 마크업에서 의도가 명확하게 드러난다.
2. flex 자식으로 `align-self: stretch`를 적용하면 strip 높이에 자동으로 맞춰진다.
3. nth-child 기반 방식보다 지표 개수 변경 시 안정적이다.

## ADR-002: 오류율 경고 표현 방식

### 상태
**결정됨** (2026-04-19)

### 배경
오류율이 임계값(5%)을 초과할 때 사용자에게 시각적으로 알려야 한다.
경고 색상을 어떻게 적용할지 결정이 필요하다.

### 고려한 옵션

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| A | `.stat-value.is-alert` 변형 클래스 | JS로 동적 토글 용이, 기존 `.stat-card.active` 패턴과 일관성 | — |
| B | CSS custom property `color: var(--red)` inline style | 즉시 적용 | 인라인 스타일은 유지보수 취약, 디자인 토큰 우회 |
| C | 별도 `.stat-card--alert` 카드 클래스 | 카드 전체 강조 가능 | 범위가 지나치게 넓어 다른 지표와 혼동 가능 |

### 결정
옵션 A — JS에서 `statValue.classList.toggle('is-alert', rate > 5)` 패턴으로 적용한다.

### 이유
1. 기존 `.stat-card.active` 토글 패턴과 동일한 방식으로 일관성을 유지한다.
2. `var(--red)` 토큰을 CSS 클래스 내에서 참조하므로 하드코딩 색상이 없다.
3. 임계값을 JS 상수로 관리하면 향후 변경이 용이하다.

## ADR-003: 아이콘 표현 방식

### 상태
**결정됨** (2026-04-19)

### 배경
비용($), P95, 오류율 등 각 지표의 성격을 아이콘으로 표현할 때 이모지 vs 텍스트 label 중 선택이 필요하다.

### 고려한 옵션

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| A | 이모지 아이콘 (💰🎯⚡❌) | 직관적 | OS/브라우저마다 렌더링 불일치, 크기 제어 어려움 |
| B | 텍스트 label (cost, saved, p95, err) | 렌더링 일관성 보장, 크기·색상 CSS 제어 가능 | 덜 직관적이나 충분히 명확 |
| C | SVG 아이콘 | 고품질 | 파일 추가·관리 비용 발생 |

### 결정
옵션 B — `stat-label`에 텍스트 label만 사용한다. 값 자체에 `$` 접두사·`%` 접미사 등 맥락 정보를 포함한다.

### 이유
1. 기존 strip의 "총 세션", "총 요청" 등 텍스트 label 패턴과 일관성을 유지한다.
2. 이모지 렌더링 불일치 문제를 원천 차단한다.
3. 추가 에셋(SVG) 없이 구현 가능하다.

## ADR-004: 데이터 미수신 초기값 처리

### 상태
**결정됨** (2026-04-19)

### 배경
서버 응답에 새 필드(cost_usd, cache_savings_usd, p95_duration_ms, error_rate)가 없거나
아직 지원되지 않을 때 표시할 초기값을 정해야 한다.

### 결정
초기값 `--` (em dash 두 개)를 HTML에 하드코딩하고, JS에서 필드가 null/undefined이면 업데이트를 건너뛴다.

### 이유
1. `--`는 "데이터 없음"을 나타내는 기존 프로젝트 관례(e.g. `statAvgDuration` 초기값)와 일치한다.
2. 서버가 필드를 아직 반환하지 않아도 UI가 깨지지 않는다.
3. 0과 미수신을 구분할 수 있다 (`$0.00` vs `--`).
